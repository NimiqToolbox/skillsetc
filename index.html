When & why

Pick the primitive that actually solves the problem.

Four worked examples — what each primitive is for, why the alternatives fall short, and the smallest snippet that does the job.

  1. 01 Why a hook?

    Block dangerous bash commands before they run.

    Memory and skills can ask Claude to be careful — they can't enforce it. Hooks fire from the harness, not the model, so the rule is unskippable.

    Use a Hook (PreToolUse) 
    {
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash",
      "hooks": [{
        "type": "command",
        "if": "Bash(rm -rf *)",
        "command": "./.claude/hooks/confirm.sh"
      }]
    }]
  }
}
  2. 02 Why a skill?

    Reusable workflows Claude auto-picks when relevant.

    A SKILL.md's description is loaded into context so Claude can match it to the user's request. The body and bundled scripts only load on demand — pay for what you use.

    Use a Skill 
    ---
name: fix-issue
description: Fix a GitHub issue. Use when the user
  references an issue number or asks to address a bug.
allowed-tools: Bash(git add *) Bash(git commit *)
arguments: [issue-number]
---
1. Read issue $ARGUMENTS[0]
2. Implement the fix
3. Write tests
4. Commit
  3. 03 Why a subagent?

    Run verbose work without flooding your main context.

    Test logs, grep output, and file dumps would balloon the main conversation. A subagent runs the same work in an isolated window and returns only the summary you need.

    Use a Subagent 
    ---
name: test-runner
description: Run the test suite and report only the failing
  cases with file paths and error messages.
tools: Bash, Read
model: haiku
---
You are a test runner. Run `npm test`,
return *only* failing test names + error output.
Never paste full logs into your reply.
  4. 04 Why path-scoped rules?

    Different rules for different parts of the codebase.

    A monorepo with API + frontend + tests doesn't want the same conventions everywhere. Path-scoped rule files load only when the user is editing matching files — zero context cost when irrelevant.

    Use a .claude/rules/<name>.md 
    ---
paths:
  - "src/api/**/*.ts"
  - "src/server/**/*.ts"
---

# API Conventions

- All endpoints return `{ ok, data, error }`
- Use Zod for input validation
- Never log full request bodies (PII risk)
- Errors throw `HTTPError(status, code, message)`

You want to… Use this primitive

Run a verbose investigation without polluting main context Subagent
Trigger something every time a tool runs or session starts Hook
